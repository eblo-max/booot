# Лендинг «Выкуп ООО» + приём заявок в Telegram

Маленький сервис на FastAPI: отдаёт страницу (`static/index.html`) и принимает
заявку из формы на `POST /api/lead`, пересылая её в личный чат Telegram.

Отдельный процесс, с поисковым ботом из корня репозитория никак не связан.

## Как это работает

```
Форма на сайте → POST /api/lead → этот сервис (хранит токен) → Telegram → ваш личный чат
```

Токен бота **нигде в коде и в HTML нет** — только в переменных окружения. Если
сервис недоступен или не настроен, форма на сайте не теряет заявку: она откроет
Telegram и положит текст в буфер обмена (мягкий откат).

## Что понадобится

1. **BOT_TOKEN** — токен вашего бота из [@BotFather](https://t.me/BotFather)
   (тот же, что в основном проекте).
2. **LEAD_CHAT_ID** — ваш числовой Telegram ID. Узнать: напишите
   [@userinfobot](https://t.me/userinfobot), он пришлёт `Id: 123456789`.
3. Один раз напишите **/start** тому боту, чей токен используете, — иначе
   Telegram не даст боту написать вам первым.

## Деплой на Railway

1. В вашем проекте Railway: **New → GitHub Repo**, выберите этот репозиторий.
2. В настройках сервиса задайте **Root Directory** → `landing`
   (чтобы Railway собирал именно этот `Dockerfile`, а не корневой бот).
3. Во вкладке **Variables** добавьте:
   - `BOT_TOKEN` = токен бота
   - `LEAD_CHAT_ID` = ваш числовой id
   - `PORT` Railway подставит сам, трогать не нужно.
4. Дождитесь деплоя, откройте выданный Railway домен — увидите лендинг.
   Отправьте тестовую заявку: сообщение должно прийти вам в Telegram.
5. Проверка «жив ли сервис»: `https://ваш-домен/health` → `{"ok": true, "configured": true}`.
   Если `"configured": false` — не заданы переменные.

Свой домен подключается в Railway → **Settings → Networking → Custom Domain**.

## Локальный запуск

```bash
cd landing
python -m venv .venv && .venv/Scripts/activate   # Windows
pip install -r requirements.txt
BOT_TOKEN=xxx LEAD_CHAT_ID=123456789 uvicorn app:app --reload
# открыть http://127.0.0.1:8000
```

## Перед публикацией — заменить заглушки

В `static/index.html` в JS-константах и ссылках остаются плейсхолдеры:

- `USERNAME` — юзернейм Telegram (ссылки «Написать», значки в шапке);
- `79990000000` — номер для WhatsApp и `tel:`;
- `mail@example.ru` — почта.

Форму они не ломают (она шлёт заявку на бэкенд), но кнопки прямых контактов
ведут в никуда, пока не заменены.

## Если лендинг переедет на отдельный домен

Когда страницу отдаёт другой хостинг, а сюда приходит только `/api/lead`,
задайте переменную `ALLOWED_ORIGIN` = адрес лендинга (например
`https://vykup-ooo.ru`) — включится CORS. При отдаче страницы и API из одного
сервиса (по умолчанию) переменная не нужна.
