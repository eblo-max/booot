# Бот поиска юридических лиц

Telegram-бот: сохраняет поисковые запросы по критериям, ходит в источник данных,
фильтрует компании и присылает в чат только новые.

Текущий статус — **этап 1**: вертикальный сценарий
`/new_search → сохранение → ручной запуск → фильтрация → дедупликация → отправка`.
Расписание, импорт Excel и выгрузка — этапы 2 и 3.

## Запуск локально

```bash
cp .env.example .env          # вписать BOT_TOKEN и ADMIN_IDS
docker compose up --build
```

Без Docker (нужен свой Postgres):

```bash
python -m venv .venv && .venv/Scripts/activate
pip install -e ".[dev]"
alembic upgrade head
python -m src.main_bot
```

## Тесты

```bash
pytest -q
```

Тесты не требуют базы: доменный слой (нормализация, фильтры, хеши) и связка
с провайдером проверяются изолированно.

## Деплой на Railway

1. `railway init`, затем добавить плагин **PostgreSQL**.
2. В сервисе бота задать переменные:
   - `BOT_TOKEN`, `ADMIN_IDS`
   - `DATABASE_URL` — reference на `${{Postgres.DATABASE_URL}}`
   - `PRIMARY_PROVIDER` — `fake` для проверки, потом `datanewton` / `checko`
   - ключи провайдеров
3. Start command берётся из `railway.json`: миграции + polling.

`DATABASE_URL` от Railway приходит в виде `postgresql://…` — конфиг сам подставляет
драйвер `asyncpg`, править ничего не нужно.

## Источники данных

Провайдеры подключаются через единый интерфейс `CompanyProvider`
(`src/providers/base.py`) и делятся на два типа:

- `mass_search=True` — отдаёт поток компаний по критериям;
- `mass_search=False` — только точечное обогащение по ИНН/ОГРН.

Если у источника нет массового поиска, бот **не имитирует** его: поднимается
`MassSearchNotSupported`, и пользователю предлагается импорт Excel/CSV.

Сейчас реализован `FakeProvider` — синтетические данные для отладки, он честно
подписывается «Тестовые данные» в каждой карточке. Боевые адаптеры (DataNewton,
Checko) добавляются после получения ключей.

## Про ОСНО

Бот никогда не пишет «ОСНО подтверждена» на основании отсутствия данных:

| Ситуация | Формулировка |
|---|---|
| источник явно подтверждает ОСНО | ОСНО подтверждена (источник: X) |
| реестр спецрежимов опрошен, режимов нет | Вероятная ОСНО — специальные режимы не обнаружены |
| найден УСН/АУСН/ЕСХН/… | название режима |
| реестр не опрошен или недоступен | Налоговый режим неизвестен |

Логика в `src/domain/tax_status.py`, покрыта тестами.

## Структура

```
src/
  domain/      критерии, нормализация, фильтр, хеши — без внешних зависимостей
  providers/   адаптеры источников за общим интерфейсом
  db/          модели SQLAlchemy 2 и репозитории
  services/    search_runner (ядро запуска), delivery (сводка + порции)
  bot/         aiogram: хендлеры, клавиатуры, мастер /new_search
migrations/    Alembic
tests/         pytest
```
